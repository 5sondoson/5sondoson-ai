# 기여 가이드 (작업 컨벤션)

이 레포는 아래 규칙으로 작업한다.

## 작업 흐름

1. `main` 에서 시작한다.
2. 작업 단위마다 이슈를 생성한다.
3. 이슈 기준으로 브랜치를 생성한다.
4. 작업 후 커밋한다.
5. PR을 생성하고 머지한다.
6. `main` 을 갱신한 뒤 다음 작업을 반복한다.

## 브랜치

- `feat/<작업 내용>`, `chore/<작업 내용>`, `fix/<작업 내용>` 형식을 사용한다.

## 커밋

- Conventional Commits 형식을 따른다: `feat:`, `chore:`, `fix:`, `test:` 등.
- 한 문장으로 작성한다.
- 논리 단위마다 커밋을 분리한다.

## 이슈 / PR

- 제목은 `feat: 작업 내용` 형식으로 쓴다.
- 내용은 핵심만 설명체로 간결하게 적는다.
- PR의 연관 이슈 칸에 `- Closes #N` 으로 적어 머지 시 이슈가 자동 종료되게 한다.
- 리뷰어는 지정하지 않는다.
- 스크린샷은 첨부하지 않는다.
- `.github` 의 PR/이슈 템플릿을 사용한다.
- 언어는 한국어로 한다.

## 로컬 테스트

- 로컬 호출은 `localhost` 대신 `127.0.0.1` 을 사용한다.
- 이유: Windows에서 `localhost` 가 IPv6(`::1`)로 먼저 풀리는데 uvicorn 은 기본적으로 IPv4(`127.0.0.1`)에 바인딩되어, `localhost` 접속이 실패할 수 있다.
